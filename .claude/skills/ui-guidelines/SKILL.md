# UI-гайдлайн vanmark-drive

Цель: быстрый, чистый, профессиональный интерфейс. Ничего «креативного» в ущерб скорости считывания: Милена живёт в этом экране весь день, водитель смотрит на телефон на улице/за рулём (на светофоре).

## Базовые правила

- Библиотека: shadcn/ui + Tailwind. Не изобретать свои контролы, где есть готовые.
- Шрифт: Inter (или системный стек). Размеры: водитель — базовый 16–18px, кнопки статусов крупные; диспетчер — базовый 14–15px, плотная информация.
- Тач-цели у водителя ≥ 48×48px, главная кнопка статуса — на всю ширину экрана, в зоне большого пальца (нижняя треть).
- Светлая тема в MVP. Без анимаций дольше 200 мс.
- Весь интерфейс на русском. Тексты короткие, без канцелярита: «Выехал», а не «Подтвердить начало движения».

## Цвета статусов (единые ВЕЗДЕ: доска, карточки, бейджи, push)

Спокойная палитра (редизайн 18.06, решение Артёма). Уходим от радуги из 9 ярких цветов к графитовой
базе с **тремя смысловыми акцентами**: цвет несёт смысл, а не украшает. Статусы различаются прежде
всего **словом-подписью**, цвет лишь подкрепляет три ключевых состояния.

| Статус | Роль | Полоса (STATUS_BAR) | Метка (STATUS_BADGE) |
|---|---|---|---|
| Новая | нейтраль | slate-300 | border-slate-300 / text-slate-600 |
| Назначена | нейтраль | slate-300 | _метку не показываем_ (Артём 24.06) |
| В работе | активна | slate-300 | **bg-blue-600 / text-white**, крупнее (size md) — насыщенная заливка, Артём 07.07 |
| Принята | нейтраль | slate-300 | border-slate-300 / text-slate-600 |
| В пути | нейтраль | slate-300 | border-slate-300 / text-slate-600 |
| На месте | нейтраль | slate-300 | border-slate-300 / text-slate-600 |
| Выполнена | **готово** | green-600 | border-green-600 / text-green-700 |
| Ждёт | **внимание** | amber-600 | border-amber-500 / text-amber-700 |
| Перенесена | нейтраль | slate-300 | border-slate-300 / text-slate-600 |
| Отменена | **сорвано** | red-600 | border-red-600 / text-red-700 |

Правила:
- Метки статусов и пропусков — **контурные** (рамка + текст, прозрачный фон), без заливок и «таблеток»
  (`rounded`, не `rounded-full`). Источник истины — `src/lib/task-ui.ts`.
- Исключения: **«В работе»** — насыщенная синяя заливка (`bg-blue-600 / text-white`) и увеличенный
  размер (`Badge size="md"`), чтобы активная задача сразу бросалась в глаза (усилено 07.07 по просьбе
  Артёма: прежний `bg-blue-50` почти терялся); **«Назначена»** — метку не показываем совсем (частый
  статус, лишний шум). Единая точка — компонент `StatusBadge` (`src/components/status-badge.tsx`) +
  `isStatusBadgeHidden`; размер бейджа задаёт проп `size` компонента `Badge`.
- Цвет = смысл: **зелёный** только «готово», **красный** только «сорвано/просрочено», **янтарный**
  только «требует действия сейчас» (Ждёт, нужен пропуск, блок «Требуют внимания»). Всё остальное — графит.
- Статус всегда показывается цветом **+ текстом** (не только цветом). Срочная задача — красная точка слева у номера.
- Пропуск: «Нужен пропуск!» — янтарный контур (сигнал внимания); «Пропуск заказан» — спокойный графит (вопрос закрыт).

## Карточка задачи (список)

Порядок считывания за 2 секунды: **№ и тип (иконка)** → **время/окно** → **суть (title)** → **адрес** → **статус-бейдж**. У водителя дополнительно крупно: индикатор пропуска («Пропуск заказан» зелёным / «Нужен пропуск!» жёлтым), кнопки «Навигатор» и «Позвонить» прямо с карточки.

## Экраны водителя — особые правила

- Один основной сценарий на экран. Никаких табов-в-табах.
- Кнопка следующего статуса — одна, большая, с текстом действия («Выехал →»). Прошедшие статусы — лента сверху.
- Оптимистичный UI: нажал — статус сразу сменился визуально; при ошибке сети — заметный, но спокойный индикатор «не отправлено, повторяю…» и автоповтор.
- Фото: кнопка открывает камеру сразу (capture), превью добавленных, удаление до отправки.
- Пустые состояния с человеческим текстом: «На сегодня задач нет 🎉» — допустимое единственное место с эмодзи.

## Экран «Сегодня» диспетчера («плотный пульт»)

- Вид — плотный диспетчерский пульт: колонки с **графитовыми шапками** (тёмный `bg-slate-900`,
  белый текст). У колонки водителя в шапке — нейтральный кружок-аватар с инициалами (АК / АП), у
  служебных колонок — иконка (`CalendarOff` / `CalendarClock`). Тело колонки белое.
- Карточки — **плотные строки** (без крупных скруглений и теней), разделены тонкой нижней границей,
  слева узкий «корешок» 3px цвета статуса. Порядок строк в карточке: № и тип + статус-метка → суть →
  адрес + время. Перетаскивание (dnd) для назначения/переназначения сохранено.
- Селект-исполнитель показывается только в пулах («Без даты», «Ближайшие 3 дня»); в колонке водителя
  он лишний — переназначение перетаскиванием в другую колонку.
- Сверху счётчики: всего / в работе / выполнено / требуют внимания (просрочки + пропуска на завтра).
  Плашки **бесцветные** (белые), цветом подсвечивается только число: «выполнено» зелёным, «требуют
  внимания» янтарным.
- Обновление данных не должно дёргать раскладку (никаких прыжков при поллинге — стабильные ключи, скелетоны только при первой загрузке).
- Создание задачи — кнопка «+ Задача» в шапке, форма в модалке/drawer, autofocus на первом поле, Enter-флоу, «создать и ещё одну».

## Перед сдачей любого UI

- [ ] Прогон в 360×740 (Android) для водительских экранов и 1366×768+ для доски.
- [ ] Все состояния: загрузка / пусто / ошибка / длинные строки (адрес в 2 строки, название ООО длинное).
- [ ] Цвета статусов из таблицы, не «похожие».
- [ ] Нет английских текстов и плейсхолдеров lorem.
