**Этап 10. KPI: экран водителя и акт-документ** — ~1 день

### Процесс акта, расценки и бонус за комплектность (Фаза 2)

Решения Артёма 18.06.2026, спека — PRD §3–§5, §12.6, §13. Электронная подпись/PDF отложены — акт бумажный (водитель прикладывает фото подписанного бланка). Цель — убрать телефонную диктовку цен и ручную писанину, не давая водителю доступ к ценам. Порядок:

**Этап 11. Типы задач и требование акта на заявке** — ~1 день
Привести справочник типов к новому списку PRD §3. Снять обязательность фото со всех типов (фото по желанию). Аддитивная миграция: per-task флаг требования акта (+ причина снятия). Галочка-исключение «акт не нужен» в форме создания; фото снято с гейта (везде по желанию). Детектор KPI «без акта» переводится на per-task флаг и оба водителя (PRD §12.1). Стоимость гарантии — в этапе 13 (часть расценки).
✅ Приёмка: диспетчер создаёт задачи нового списка; галочка «акт не нужен» снимает требование, причина видна в карточке; DONE без фото проходит на любом типе; детектор «без акта» учитывает флаг заявки; security-check.

**Этап 12. Ведомость работ у водителя + справочник работ** — 1–2 дня

### Офлайн-режим водителя (Фаза 2)

Реализовано серией коммитов в одной ветке. Только клиент водителя (Android), без новых зависимостей.

**Этап O1. Серверная идемпотентность + клиентское время** — `ProcessedAction` (аддитивная миграция); `withIdempotency` (Idempotency-Key, чужой ключ → 404); `resolveOccurredAt` (clamp X-Occurred-At); 8 мутаций водителя обёрнуты; transition/comment пишут время по occurredAt. Unit + security-check.
✅ Приёмка: повтор того же действия не даёт двойного эффекта; чужой ключ → 404; время действия — момент нажатия.

**Этап O2. Кэш чтения** — IndexedDB-обёртка; `cachedFetcher` (GET кэшируется, офлайн отдаёт сохранённое); список и карточка открываются без сети; баннеры офлайна. Unit.
✅ Приёмка: офлайн список «Мои задачи» и карточка показывают последнее загруженное.

**Этап O3. Очередь записи + синхронизатор + оверлей + конфликты** — мутации офлайн в очередь, автодосылка при связи; оверлей статуса; бейджи «ждёт отправки»; конфликт (4xx при досылке) → пометка водителю. Unit + e2e.
✅ Приёмка: офлайн «В работу»/«Завершить»/комментарий/ведомость копятся и долетают при связи; статус виден сразу.
